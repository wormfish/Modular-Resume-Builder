Single-process Express application bootstrapped from `server/index.js`: it loads `.env` via `dotenv`, mounts CORS and a JSON body parser (5 MB limit), then registers two feature routers — `/api/resumes` (`routes/resumes.js`) and `/api/blocks` (`routes/blocks.js`) — plus a `/api/health` probe. Data access is split by domain into `models/Resume.js` and `models/Block.js`, each defining a Mongoose schema and exporting the model; routes import these models directly with no service layer. Startup sequence connects to MongoDB first and only starts listening after the connection resolves. The Resume document owns an embedded `sections` array of sub-documents referencing block IDs, while Block stores its payload in a `Mixed` `content` field keyed by an enum'd `type` (`summary | experience | education | skills`). There is no authentication, validation middleware, or error-handling middleware — errors bubble up as `{ error }` JSON responses per route handler.
