- Entry point `src/main.jsx` mounts `<App />` under `React.StrictMode` and loads `print.css` for PDF output.
- `src/App.jsx` is the single source of truth: it owns all state (`blocks`, `resume`, `personalInfo`, `jobTypes`) via the `useApiSync` hook, orchestrates block CRUD, section management, drag-and-drop reordering, and JSON/PDF export/import, then renders four presentational panels — `BlockLibrary`, `ResumeCanvas`, `PropertiesPanel`, and `BlockModal`.
- State layer: `useApiSync.js` wraps each piece of state in `useLocalStorage` (keys `resume-builder-*`) so data survives reloads, then attempts an initial fetch from `/api/blocks` and `/api/resumes`; mutations are optimistic — local state updates immediately while background calls to `api/client.js` set `isOnline` on success/failure. Resume saves are debounced at 600 ms via `scheduleResumeSync`.
- API layer: `src/api/client.js` exposes typed functions (`fetchBlocks`, `createBlock`, `updateBlock`, `deleteBlock`, `fetchResumes`, `saveResume`, `deleteResume`, `checkHealth`) over a shared `request(path, options)` helper that POSTs/PUTs/DELETEs against `/api` and throws on non-2xx responses.
- Rendering model: `BLOCK_SCHEMA` in `utils/constants.js` declares each block type's fields + a `render(b)` function; `ResumeCanvas` looks up the schema by `block.type` and delegates rendering to `ResumeBlock`, keeping new block types declarative rather than imperative.
- Templates are similarly declared as `{ name, description, className }` entries in `TEMPLATES` and applied via CSS class names on the page container.
- Side-effect hooks: `useExportPdf.js` (html2canvas + jsPDF), `useJsonExportImport.js` (FileReader-based import/export of full resume / blocks / personal info), `useLocalStorage.js` (localStorage wrapper).
- Dependency direction: components → hooks → api/client → browser APIs; constants are pure data consumed everywhere. No cross-component imports beyond what App composes.
